# 11주차 실습 정리
## 오늘 한 것
- PyInstaller 설치 및 빌드
- resource_path() 함수 추가
- --add-data 옵션으로 에셋 포함
- .exe 실행 확인

## resource_path() 를 써야 하는 이유
실행 파일은 실행 시 내부 파일들을 가상의 임시 폴더에 압축 해제하며 작동하기 때문에, 
기존의 일반적인 파일 경로로는 이미지나 사운드 같은 에셋을 찾을 수 없습니다. 
resource_path() 함수는 프로그램이 실행되는 실제 위치를 실시간으로 파악하여 에셋의 정확한 경로를 연결해 주는 역할을 합니다. 
이 함수를 사용해야만 빌드 후 파일 로드 오류로 인해 프로그램이 강제 종료되는 현상을 막을 수 있습니다.
## 빌드 명령어
python -m pip install pyinstaller
## AI 활용 내역
PyInstaller 설치 과정에서 발생하는 경로 및 환경 변수 오류를 AI와의 문답을 통해 즉시 해결하였습니다. 
특히 윈도우 환경의 빌드 규칙인 세미콜론(;) 사용법과 게임 실행에 필수적인 --noconsole 옵션의 의미를 정확히 파악하여 적용했습니다. 
이를 통해 단순 코딩을 넘어 배포 가능한 프로그램으로 완성하는 전체 프로세스를 학습하였습니다.
